Error Handling

                
The SkyWatch API uses conventional HTTP response codes to indicate success or failure.

                
                

                    
HTTP Status Codes

                    
                    

                        

                            200
                            OK
                        

                        
Everything worked as expected.

                    

                    
                    

                        

                            400
                            Bad Request
                        

                        
The request was unacceptable, often due to missing a required parameter.

                    

                    
                    

                        

                            401
                            Unauthorized
                        

                        
No valid API key provided.

                    

                    
                    

                        

                            403
                            Forbidden
                        

                        
The API key doesn't have permissions to perform the request.

                    

                    
                    

                        

                            404
                            Not Found
                        

                        
The requested resource doesn't exist.

                    

                    
                    

                        

                            429
                            Too Many Requests
                        

                        
Too many requests hit the API too quickly. Rate limiting is active.

                    

                    
                    

                        

                            500
                            Internal Server Error
                        

                        
Something went wrong on our end.

                    

                    
                    

                        

                            503
                            Service Unavailable
                        

                        
The server is temporarily unable to handle the request.

                    

                

                
                
Error Response Format

                
All errors return JSON with the following structure:

                
{
  "error": "error_code",
  "message": "Human-readable error description",
  "details": {
    "parameter": "Additional error context"
  }
}

                
                
Example Error Response

                
{
  "error": "Invalid API key",
  "message": "The provided API key is invalid or has been revoked.",
  "details": {
    "code": "invalid_api_key",
    "timestamp": "2023-12-15T10:30:00Z"
  }
}

Data Schemas

                
Detailed schemas for all API objects and responses.

                
                

                    
Sighting Object

                    
Represents a UFO sighting report from the NUFORC database.

                    
                    

                        

                            
id integer

                            
Unique identifier for the sighting

                        

                        
                        

                            
date_time string

                            
ISO 8601 timestamp of when the sighting occurred

                        

                        
                        

                            
city string

                            
City where the sighting was reported

                        

                        
                        

                            
state string

                            
US state abbreviation (2 letters)

                        

                        
                        

                            
shape string

                            
Reported shape of the UFO (e.g., "disk", "triangle", "light")

                        

                        
                        

                            
duration string

                            
How long the sighting lasted

                        

                        
                        

                            
summary string

                            
Brief summary of the sighting

                        

                        
                        

                            
text string

                            
Full detailed report of the sighting

                        

                        
                        

                            
posted string

                            
ISO 8601 timestamp of when the report was posted

                        

                        
                        

                            
latitude number

                            
Latitude coordinate (if available)

                        

                        
                        

                            
longitude number

                            
Longitude coordinate (if available)

sightings array[Sighting]

                            
Array of sighting objects

                        

                        
                        

                            
total integer

page integer

                            
Current page number (1-indexed)

                        

                        
                        

                            
per_page integer

                            
Number of items per page

                        

                        
                        

                            
pages integer

                            
Total number of pages

Rate Limit Headers

                
Every API response includes rate limit information in the headers:

                
                
HTTP/1.1 200 OK
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 299
X-RateLimit-Reset: 1640995200
X-RateLimit-Used: 1

                
                

                    

                        
X-RateLimit-Limit

                        
Maximum requests allowed per hour for your tier

                    

                    
                    

                        
X-RateLimit-Remaining

                        
Number of requests remaining in the current hour

                    

                    
                    

                        
X-RateLimit-Reset

                        
Unix timestamp when the rate limit resets

                    

                    
                    

                        
X-RateLimit-Used

                        
Number of requests used in the current hour

                    

                

                
                
Handling Rate Limits

                
When you exceed your rate limit, the API returns a 429 Too Many Requests response:

                
                
{
  "error": "Rate limit exceeded",
  "message": "Rate limit of 300 requests per hour exceeded.",
  "rate_limit": 300,
  "retry_after": 3600
}

                
                

                    
Best Practices

                    
    
                        
  • Monitor the rate limit headers in your responses
    • 
                        
  • Implement exponential backoff when approaching limits
    • 
                        
  • Cache responses when possible to reduce API calls
    • 
                        
  • Use pagination efficiently to avoid large responses
    • 
                        
  • Consider upgrading your tier if you regularly hit limits
